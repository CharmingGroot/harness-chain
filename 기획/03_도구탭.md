# 03. 도구 탭 (Tools)

## 개요

도구 탭은 에이전트가 사용할 수 있는 도구를 관리하는 화면이다. 도구는 에이전트 루프에서 LLM이 직접 호출하는 실행 단위다. 현재 SQL 관련 도구 5종이 내장(`builtin`)되어 있으며, 이메일·Slack·Notion·웹훅·파일 저장은 "준비 중" 상태다.

---

## 도구 종류 및 상태

| 도구 | 카테고리 | 아이콘 | 상태 | 비고 |
|---|---|---|---|---|
| SQL 쿼리 (5종) | database | 🗄 | 내장 | execute_query, explain_query, get_schema 등 |
| 이메일 발송 | notification | 📧 | 준비 중 | SMTP / SendGrid |
| Slack 메시지 | notification | 💬 | 준비 중 | 채널 / DM |
| Notion 업데이트 | integration | 📊 | 준비 중 | 페이지 / DB 작성 |
| 웹훅 호출 | integration | 🔔 | 준비 중 | 임의 HTTP POST |
| 파일 저장 | storage | 📁 | 준비 중 | 로컬 / S3 |

레지스트리 API(`GET /api/registry`)에서 `comingSoon: true`인 도구는 서브에이전트 빌더의 도구 선택 목록에서 제외된다.

---

## ITool 인터페이스

```typescript
interface ITool {
  readonly name: string;
  readonly description: string;
  readonly inputSchema: Record<string, unknown>; // JSON Schema
  execute(input: unknown): Promise<ToolResult>;
}

interface ToolResult {
  success: boolean;
  data?: unknown;
  error?: string;
  text: string;  // LLM에 전달되는 텍스트 표현
}
```

## 내장 SQL 도구 상세

모두 `lib/tools/sql-tools.ts`에 구현되어 있으며, `PostgreSQLSource` 생성 시 pg.Pool을 주입받는다.

### execute_query

SQL 쿼리를 실행하고 결과를 반환한다.

```typescript
inputSchema: {
  sql: string;    // 실행할 SQL (필수)
  limit?: number; // 최대 반환 행 수, 기본 200, 최대 2000
}
```

- SELECT 쿼리에 LIMIT이 없으면 자동으로 추가한다.
- 결과는 ASCII 테이블 형식 문자열(`text`)로 LLM에 전달된다.
- 최대 30행까지 테이블로 포맷, 초과분은 "... (N more rows)" 요약.

### explain_query

쿼리 실행 계획을 반환한다.

```typescript
inputSchema: {
  sql: string;       // 분석할 SQL (필수)
  analyze?: boolean; // 실제 실행 후 측정 여부, 기본 false
}
```

- `EXPLAIN (FORMAT JSON)` 또는 `EXPLAIN (ANALYZE, BUFFERS, FORMAT JSON)` 실행.
- Node Type, Total Cost, Plan Rows, Index Name 요약 반환.

### get_schema

DB 스키마(테이블, 컬럼 목록, row count)를 반환한다.

```typescript
inputSchema: {
  tables?: string[]; // 특정 테이블만 조회, 미지정 시 전체
}
```

- `information_schema.tables`, `information_schema.columns` 조인 쿼리.
- `pg_stat_user_tables`에서 실시간 행 수(n_live_tup) 조회.
- 에이전트 시스템 프롬프트에서 **쿼리 작성 전 반드시 호출**하도록 지시.

### get_table_sample

테이블 샘플 데이터를 반환한다.

```typescript
inputSchema: {
  table: string;  // 테이블 이름 (필수, 영문자·숫자·언더스코어만 허용)
  rows?: number;  // 반환 행 수, 기본 5, 최대 20
}
```

- SQL 인젝션 방지: 정규식 `/^[a-zA-Z_][a-zA-Z0-9_]*$/`으로 테이블명 검증.

### get_stats

테이블 크기, seq_scan vs idx_scan 비율 등 DB 성능 통계를 반환한다.

```typescript
inputSchema: {
  table?: string; // 특정 테이블, 미지정 시 전체 (최대 20개, live_rows 내림차순)
}
```

- `pg_stat_user_tables`에서 live_rows, seq_scan, idx_scan, total_size 조회.

## 도구 등록 방식 (CA 아키텍처)

HarnessChain의 `ITool`은 `@charming_groot/core`의 `ITool`과 인터페이스가 다르다. `HarnessToolAdapter` (`lib/ca-adapter.ts`)가 변환을 담당한다.

```
HarnessTool (text 기반)
    ↓ HarnessToolAdapter
CATool (CA 표준 인터페이스)
    ↓ Registry.register()
AgentLoop에서 호출 가능
```

**HarnessToolAdapter 역할:**
- `inner.inputSchema`의 `properties`/`required`를 CA `ToolDescription.parameters` 형식으로 변환.
- `inner.execute(params)` → `{ success, output: text, error }` 형식으로 래핑.
- `requiresPermission = false` (실행 시 사용자 승인 불필요).

## 레지스트리 API

```
GET /api/registry
```

**응답:**
```json
{
  "sources": [
    { "id": "pg-sandbox", "name": "pg-sandbox", "type": "postgresql", ... }
  ],
  "tools": [
    { "id": "execute_query", "name": "execute_query", "category": "database", "description": "SQL 쿼리 실행" },
    { "id": "explain_query", "name": "explain_query", "category": "database", "description": "쿼리 실행계획 분석" },
    { "id": "get_schema",    "name": "get_schema",    "category": "database", "description": "DB 스키마 조회" },
    { "id": "get_table_sample", ... },
    { "id": "get_stats",    ... },
    { "id": "http_request", ..., "comingSoon": true },
    { "id": "slack_notify", ..., "comingSoon": true },
    { "id": "email_send",   ..., "comingSoon": true },
    { "id": "write_file",   ..., "comingSoon": true }
  ],
  "subAgents": [ ... ]  // 파일 스토어에서 동적으로 로드
}
```

소스·도구 목록은 코드에 정적 정의, 서브에이전트는 `data/subagents.json`에서 동적 로드.
