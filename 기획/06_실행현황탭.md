# 06. 실행 현황 탭 (Observability)

## 개요

실행 현황 탭은 하네스 실행 이력을 조회하고 상태를 모니터링하는 화면이다. `GET /api/queue` 엔드포인트에서 데이터를 가져와 통계 지표와 실행 목록 테이블을 표시한다. 화면 진입 시 1회 조회하며, 실시간 폴링은 없다.

---

## 화면 구성

### 통계 카드 (4개)

| 지표 | 색상 | 계산 방식 |
|---|---|---|
| 전체 | `var(--text-primary)` | 전체 run 수 |
| 대기 중 | `var(--accent)` | status === "pending" |
| 실행 중 | `#22c55e` | status === "running" AND Redis에 있음 |
| 완료 | `var(--text-tertiary)` | status === "completed" |

실패(failed)는 별도 카드 없이 목록에서 빨간색으로 표시된다.

### 실행 목록 테이블

최근 50개 실행을 역순(최신 먼저)으로 표시한다.

| 컬럼 | 내용 |
|---|---|
| 하네스 | `harnessName` (하네스 ID → 이름 매핑) |
| 트리거 | 현재 항상 `"manual"` |
| 예약 시각 | `startedAt` (HH:MM 형식) |
| 소요 시간 | `durationMs / 1000` 초 단위 (소수점 1자리) |
| 상태 | 상태별 색상 텍스트 |

**상태 색상:**

| 상태 | 색상 | 레이블 |
|---|---|---|
| scheduled | `var(--accent)` | 예약됨 |
| running | `#22c55e` | 실행 중 |
| completed | `var(--text-tertiary)` | 완료 |
| failed | `#ef4444` | 실패 |

실패 시 `error` 필드 내용이 hover tooltip으로 표시된다.

## API: GET /api/queue

```typescript
// 응답 구조
{
  jobs: {
    id: string;
    harnessId: string;
    harnessName: string;
    status: "pending" | "running" | "completed" | "failed";
    trigger: "manual";
    startedAt: string;
    finishedAt: string | null;
    durationMs: number | null;
    error: string | null;
    meta: {
      toolCallCount?: number;
      iterations?: number;
      sourcesUsed?: string[];
    } | null;
  }[];
  metrics: {
    totalJobs: number;
    running: number;    // Redis 확인된 실행 중
    pending: number;
    completed: number;
    failed: number;
    avgDurationMs: number;
  };
}
```

## 상태 결정 로직

파일 스토어의 `status`와 Redis 활성 Set을 교차 검증하여 실제 상태를 결정한다.

```
run.status === "running" && activeInRedis.has(run.id)
  → "running"  (정상 실행 중)

run.status === "running" && !activeInRedis.has(run.id)
  → "failed"   (고아 run — 서버 재시작으로 중단됨)
  → error: "서버 재시작으로 인해 중단됨"

run.status === "completed" | "failed" | "pending"
  → 그대로 사용
```

**통계의 failed 계산:**
```typescript
failed: runs.filter(r =>
  r.status === 'failed' ||
  (r.status === 'running' && !activeInRedis.has(r.id))
).length
```

## 데이터 저장 구조

### 파일 스토어 (`data/runs.json`)

```typescript
interface HarnessRun {
  id: string;           // "run_{timestamp}_{random5}"
  harnessId: string;
  status: RunStatus;    // "pending" | "running" | "completed" | "failed"
  startedAt: string;    // ISO 8601
  completedAt?: string;
  durationMs?: number;
  report?: string;      // 최종 마크다운 리포트
  error?: string;       // 실패 시 오류 메시지
  meta?: {
    toolCallCount?: number;
    iterations?: number;
    sourcesUsed?: string[];
  };
}
```

### Redis 키 스키마

```
hc:active               Set   — 현재 실행 중인 runId 목록
hc:run:{runId}          Hash  — { harnessId, status, startedAt }
                                TTL: 30분 (1800초)
```

실행 시작 시 파이프라인으로 원자적 처리:
```
HSET hc:run:{runId} harnessId {id} status running startedAt {iso}
EXPIRE hc:run:{runId} 1800
SADD hc:active {runId}
```

실행 종료(완료/실패) 시:
```
DEL hc:run:{runId}
SREM hc:active {runId}
```

## 고아 run 복구

서버 시작 시 `instrumentation.ts`에서 `recoverOrphanedRuns()`를 호출한다 (Next.js instrumentation hook).

```typescript
// 파일 스토어에서 status==="running"인 run 목록 조회
// Redis에 없으면(TTL 만료 또는 서버 재시작) → failed 처리
async function recoverOrphanedRuns(): Promise<number>
```

Redis 연결 실패는 비치명적으로 처리된다 — Redis가 없어도 실행 이력 조회와 실행 자체는 계속 동작한다.

## 평균 실행 시간

```typescript
avgDurationMs: (() => {
  const done = allRuns.filter(r => r.durationMs != null);
  if (!done.length) return 0;
  return Math.round(
    done.reduce((s, r) => s + (r.durationMs ?? 0), 0) / done.length
  );
})()
```

현재 UI에는 표시되지 않지만 API 응답에 포함되어 있다.
