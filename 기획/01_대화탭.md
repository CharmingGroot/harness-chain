# 01. 대화 탭 (Chat)

## 개요

대화 탭은 HarnessChain의 주 인터페이스다. 사용자는 자연어로 분석 요청을 보내고, 시스템은 SSE 스트리밍으로 실시간 진행 상태와 최종 리포트를 반환한다. 여러 대화를 세션 단위로 관리하며, 각 세션은 독립적인 메시지 이력을 갖는다.

---

## 세션 관리

### 세션 구조

```typescript
interface Session {
  id: string;       // "sess_{timestamp}"
  name: string;     // "세션 1", "세션 2" ...
  createdAt: string;
}
```

- 세션은 브라우저 `localStorage`에 `hc_sessions` 키로 저장된다.
- 각 세션의 메시지는 `hc_msgs_{sessionId}` 키로 분리 저장된다.
- 서버 재시작(페이지 새로고침) 시 `streaming` 상태인 메시지는 자동으로 `error` 처리된다.

### 세션 작업

| 작업 | 동작 |
|---|---|
| 새 세션 생성 | `+ 새 세션` 버튼 → `sess_{Date.now()}` ID 생성, 대화 탭으로 전환 |
| 세션 전환 | 좌측 사이드바에서 세션 클릭 |
| 세션 삭제 | 항목 hover 시 나타나는 ✕ 버튼 (스트리밍 중에는 비활성) |
| 미리보기 | 가장 최근 사용자 메시지 앞 28자 표시 |

---

## 메시지 구조

```typescript
interface ChatMessage {
  id: string;                    // "msg_{timestamp}"
  role: "user" | "assistant";
  content: string;               // 최종 리포트 (마크다운)
  logs?: LogEntry[];             // SSE 이벤트 로그 (도구 호출 이력)
  status?: "streaming" | "done" | "error";
  createdAt: string;
  attachedItems?: PaletteItem[]; // / 커맨드로 첨부된 항목
}
```

**LogEntry 종류:**

| kind | 표시 내용 |
|---|---|
| `source_check` | 소스 연결 확인 메시지 |
| `source_selected` | 사용 가능 소스 목록 |
| `tool_call` | 호출된 도구 이름 + 입력값 앞 90자 |
| `tool_result` | 도구 실행 결과 성공/실패 + 미리보기 120자 |
| `thinking` | LLM 내부 추론 텍스트 |
| `error` | 오류 메시지 |

## 모델 선택

입력창 하단 우측의 드롭다운에서 LLM 모델을 선택한다. 선택한 모델은 세션 전체에 적용되며, 전송 시 `POST /api/analyze` 요청 body에 `model` 필드로 전달된다.

**지원 모델 목록:**

| 제공자 | 모델 ID | 레이블 |
|---|---|---|
| Anthropic | `claude-opus-4-6` | Claude Opus 4.6 |
| Anthropic | `claude-sonnet-4-6` | Claude Sonnet 4.6 |
| Anthropic | `claude-haiku-4-5-20251001` | Claude Haiku 4.5 |
| Anthropic | `claude-3-5-sonnet-20241022` | Claude 3.5 Sonnet |
| Anthropic | `claude-3-5-haiku-20241022` | Claude 3.5 Haiku |
| Anthropic | `claude-3-opus-20240229` | Claude 3 Opus |
| OpenAI | `gpt-5.4-mini` | GPT-5.4 mini (기본값) |
| OpenAI | `gpt-4.1-mini` | GPT-4.1 mini |
| OpenAI | `gpt-4o` | GPT-4o |
| OpenAI | `o3-mini` | o3-mini |
| OpenAI | `o1` | o1 |

모델 ID가 `gpt-` 또는 `o1`, `o3`으로 시작하면 OpenAI 프로바이더를 사용하고, 그 외에는 Anthropic 프로바이더를 사용한다.

## / 커맨드 팔레트

입력창에서 `/`를 입력하면 팔레트가 팝업된다. `/`  이후 텍스트로 이름/설명 필터링이 적용된다.

### 팔레트 항목 종류

| kind | 아이콘 | 소스 |
|---|---|---|
| `harness` | ⛓ | `GET /api/harnesses` |
| `tool` | ⚙ | `GET /api/registry` (comingSoon 제외) |
| `subagent` | 🤖 | `GET /api/registry` |

### 키보드 탐색

| 키 | 동작 |
|---|---|
| `↓` / `↑` | 항목 이동 |
| `Enter` | 선택 (첨부) |
| `Escape` | 팔레트 닫기 |

### 첨부 동작

항목을 선택하면:
1. 입력창의 `/` 이후 텍스트가 삭제된다.
2. 입력창 위에 칩(chip) 형태로 첨부 항목이 표시된다.
3. 메시지 전송 시 첨부 항목 이름이 `[하네스: 이름]`, `[도구: 이름]`, `[에이전트: 이름]` 형식으로 본문 앞에 조합된다.

`+` 버튼을 클릭해도 동일한 팔레트를 열 수 있다.

## SSE 스트리밍

### 흐름

```
POST /api/analyze
  Content-Type: application/json
  { query: string, model?: string }

Response:
  Content-Type: text/event-stream
  data: { type: "source_check", message: "..." }
  data: { type: "source_selected", sources: ["pg-sandbox 금융 DB"] }
  data: { type: "tool_call", tool: "get_schema", input: "..." }
  data: { type: "tool_result", tool: "get_schema", success: true, preview: "..." }
  data: { type: "tool_call", tool: "execute_query", input: "..." }
  data: { type: "tool_result", tool: "execute_query", success: true, preview: "..." }
  ... (maxIterations=25)
  data: { type: "report", report: "# 분석 결과\n...", meta: { sourcesUsed, toolCallCount, iterations } }
```

### 클라이언트 파싱

`ReadableStream` API로 스트림을 읽고, `\n\n` 구분자로 SSE 청크를 분리한다. `data: ` 접두사를 제거한 후 JSON 파싱한다. `report` 이벤트가 도착하면 `status: done`으로 전환하고, 스트림이 종료되어도 `report`가 없으면 `status: error`로 처리한다.

## 채팅 버블 UI

### 사용자 메시지
- 우측 정렬, 파란 배경

### 어시스턴트 메시지
- 도구 호출 횟수 표시 버튼 (클릭 시 로그 펼치기/접기)
- 스트리밍 중: 마지막 tool_call/tool_result 실시간 표시
- 최종 리포트: `renderMarkdown()`으로 HTML 변환 후 `dangerouslySetInnerHTML` 렌더링

## 실행 모니터 패널 (우측)

헤더 우측 `▣` 버튼으로 열린다. 현재 진행 중이거나 로그가 있는 어시스턴트 메시지 최대 10개를 역순으로 표시한다.

**MonitorCard 구성:**
- 상단: 상태(스피너/✓/✗), 첨부 하네스명 또는 세션명, 도구 호출 횟수
- 하네스가 첨부된 경우: 단계 그래프 (완료된 단계 녹색, 실행 중 단계 강조 + 애니메이션)
- 하단: 마지막 6개 로그 라인

## 초기 상태 (빈 세션)

세션이 없을 때는 "새 세션 시작" 버튼만 표시한다. 세션이 있지만 메시지가 없을 때는 등록된 하네스 최대 3개를 빠른 실행 버튼으로 표시한다.
