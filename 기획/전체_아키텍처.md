# HarnessChain — 전체 아키텍처

## 시스템 개요

HarnessChain은 AI 오케스트레이터 기반의 데이터 분석 자동화 플랫폼이다. 사용자는 자연어로 분석 요청을 입력하고, 시스템은 LLM 에이전트 루프를 통해 데이터 소스에서 정보를 수집·분석한 뒤 마크다운 리포트를 생성한다. 정기적으로 반복할 분석 작업은 "하네스"로 저장해 크론 스케줄에 따라 자동 실행할 수 있다.

## 기술 스택

| 레이어 | 기술 |
|---|---|
| 프런트엔드 | Next.js 15 (App Router), React 19, Tailwind CSS |
| 백엔드 API | Next.js Route Handlers (서버 컴포넌트) |
| AI 오케스트레이션 | `@charming_groot/core` — Registry, EventBus, AgentLoop |
| LLM 프로바이더 | Anthropic Claude (claude-sonnet-4-6 기본), OpenAI GPT |
| 서브에이전트 | `@charming_groot/agent` — SubAgentTool, ToolDispatcher |
| 데이터 소스 | PostgreSQL 16 (pg 드라이버, 커넥션 풀) |
| 상태 저장 | 파일 시스템 (JSON), Redis (실행 중 상태 TTL) |
| 런타임 모니터링 | SSE (Server-Sent Events) 스트리밍 |
| 패키지 매니저 | pnpm workspace |

## 데이터 흐름

### 1. 대화 쿼리 (SSE)

```
브라우저
  → POST /api/analyze { query, model }
  → Orchestrator.run(query)
      → PostgreSQLSource.ping()         # 소스 연결 확인
      → Registry에 도구·서브에이전트 등록
      → AgentLoop.run(request)
          → LLM 호출 (Claude / OpenAI)
          → tool:start 이벤트  → SSE: { type: "tool_call" }
          → ITool.execute()
          → tool:end 이벤트    → SSE: { type: "tool_result" }
          → 반복 (maxIterations=25)
      → SSE: { type: "report", report, meta }
  → 브라우저: ChatBubble에 마크다운 렌더링
```

### 2. 하네스 실행 (비동기)

```
브라우저
  → POST /api/harnesses/{id}/run
  → executeHarness(id)
      → createRun(harnessId)             # 파일 스토어 pending
      → markRunning(runId)               # Redis SADD hc:active
      → buildPromptFromHarness(harness)  # 단계 → 프롬프트 변환
      → Orchestrator.run(prompt)
      → updateRun(runId, { status: completed, report, meta })
      → clearRunning(runId)             # Redis SREM
```

### 3. 크론 스케줄 실행

```
서버 시작 시 scheduleHarness(id, cronExpression) 등록
  → setInterval(cronToMs(cron))
  → executeHarness(harnessId) (위 흐름과 동일)
```

## 상태 저장 전략

| 데이터 | 저장소 | 파일/키 |
|---|---|---|
| 서브에이전트 정의 | 파일 (JSON) | `data/subagents.json` |
| 하네스 정의 | 파일 (JSON) | `data/harnesses.json` |
| 실행 이력 (영구) | 파일 (JSON) | `data/runs.json` |
| 실행 중 상태 (임시) | Redis | `hc:active` Set, `hc:run:{id}` Hash (TTL 30분) |
| 대화 세션·메시지 | 브라우저 localStorage | `hc_sessions`, `hc_msgs_{sessionId}` |

파일 스토어는 영구 감사 기록, Redis는 "현재 실행 중" 상태의 단일 진실 공급원(source of truth)이다. 서버 재시작 시 Redis에서 사라진 실행 레코드는 자동으로 `failed` 처리된다 (`recoverOrphanedRuns()`).
