# 05. 하네스 탭 (Harnesses)

## 개요

하네스 탭은 반복적인 데이터 분석 작업을 "하네스"로 정의하고 관리하는 화면이다. 하네스는 실행 단계(steps) 목록과 스케줄을 가지며, 수동 또는 크론 스케줄에 따라 자동 실행된다. 빌드 모드에서는 자연어 설명으로 분석을 즉시 수행한 뒤 결과를 하네스로 저장할 수 있다.

---

## 하네스 구조

```typescript
interface HarnessDef {
  id: string;          // "h_{timestamp}_{random4}"
  name: string;
  description: string;
  steps: HarnessStep[];
  schedule: Schedule;  // { type: "once" } | { type: "cron"; cron: string }
  createdAt: string;
  updatedAt: string;
}

interface HarnessStep {
  id: string;
  kind: "subagent" | "tool" | "source";
  ref: string;         // 서브에이전트 ID, 도구 이름, 소스 ID
  label?: string;      // 표시용 이름 (없으면 ref 사용)
}

type Schedule =
  | { type: "once" }                    // 즉시 실행 (일회성)
  | { type: "cron"; cron: string };     // 크론 표현식
```

---

## 화면 구성

탭 내부는 두 가지 모드로 전환된다.

| 모드 | 설명 |
|---|---|
| 하네스 목록 | 저장된 하네스 카드 목록 |
| + 새로 빌드 | AI로 분석 후 저장 |

하네스가 없으면 자동으로 빌드 모드로 시작한다.

---

## 하네스 목록 (list 모드)

각 하네스는 카드로 표시된다.

| 표시 항목 | 내용 |
|---|---|
| 아이콘 | 크론이면 ⏰, 즉시 실행이면 ▷ |
| 이름, 설명 | |
| 스케줄 | "크론: {cron}" 또는 "즉시 실행" |
| 단계 수 | "{N}단계" |
| 생성일 | `ko-KR` 형식 날짜 |
| ▶ 실행 버튼 | `POST /api/harnesses/{id}/run` 호출 |

실행 버튼 클릭 시 해당 하네스 ID가 `runningIds` Set에 추가되어 버튼이 "실행 중..." 비활성 상태로 변경된다.

## 빌드 모드 (build 모드)

### 컨텍스트 표시

빌드 화면 상단에 현재 사용 가능한 리소스 수를 표시한다.

```
📦 소스 {N}개   ⚙ 도구 {N}개   🤖 서브에이전트 {N}개
```

### 템플릿

| 템플릿 | 설명 |
|---|---|
| 📊 VIP 이탈 위험 일일 알림 | 매일 오전 9시, 이탈 위험 VIP 분석 |
| 🚨 이상 거래 실시간 탐지 | 매 시간, 이상 거래 패턴 탐지 |
| 📉 대출 부실 주간 보고 | 매주 월요일, 연체 현황 분석 |

### 빌드 흐름

1. 사용자가 자동화할 업무를 자연어로 입력한다.
2. `POST /api/analyze`를 호출하여 SSE 스트리밍으로 분석을 수행한다.
3. 진행 중에는 로그 패널에 도구 호출 이력이 실시간 표시된다.
4. `report` 이벤트 수신 시 분석 결과(마크다운)가 표시된다.
5. 메타 정보(툴 호출 횟수, 반복 횟수, 소요 시간)가 상단에 표시된다.
6. "하네스로 저장" 버튼 클릭 시 `POST /api/harnesses`로 저장된다.

빌드 모드는 `/api/analyze`(대화 탭과 동일한 엔드포인트)를 사용하며, 분석 완료 후 저장 시 steps는 빈 배열(`[]`), schedule은 `{ type: "once" }`로 저장된다.

---

## 하네스 저장 (POST /api/harnesses)

```
POST /api/harnesses
{
  name: string;          // processDesc 앞 40자
  description: string;   // processDesc 앞 100자
  steps: [];             // 현재 빌드 시 빈 배열로 저장
  schedule: { type: "once" }
}
→ 201 Created: HarnessDef
```

저장 후 `onSaved()`가 호출되어 하네스 목록을 갱신하고 대화 탭으로 전환된다.

## 하네스 실행 (POST /api/harnesses/{id}/run)

### API 동작

```
POST /api/harnesses/{id}/run
Body: { prompt?: string }  // 없으면 buildPromptFromHarness()로 자동 생성
→ 202 Accepted: HarnessRun (status: running 또는 pending)
```

**Fire-and-forget 패턴:** 응답은 200ms 타임아웃 후 즉시 반환된다. 실제 실행은 백그라운드에서 계속된다. 타임아웃 내에 완료되면 완료된 run 객체를 반환하고, 타임아웃되면 `{ id: "run_pending", status: "running" }` 임시 객체를 반환한다.

### 프롬프트 자동 생성 (buildPromptFromHarness)

steps가 정의된 하네스의 경우 단계 목록을 자연어 프롬프트로 변환한다.

```
하네스 '{name}'를 실행합니다.
목표: {description}

실행 단계:
1. 서브에이전트 '{ref}'에게 위임   ← kind: "subagent"
2. 도구 '{ref}' 실행               ← kind: "tool"
3. 소스 '{ref}' 조회               ← kind: "source"

위 단계를 순서대로 실행하고 최종 리포트를 작성하세요.
```

## 크론 스케줄러

`lib/executor.ts`에 인-프로세스 cron 스케줄러가 구현되어 있다.

```typescript
// 크론 → 밀리초 변환 (5필드 cron 간이 파싱)
function cronToMs(cron: string): number {
  // "* * * * *" → 60,000ms (매분)
  // "0 * * * *" → 3,600,000ms (매시간)
  // "0 9 * * *" → 86,400,000ms (매일)
}

// 스케줄 등록
scheduleHarness(harnessId, "0 9 * * *");
// → setInterval(executeHarness, 86400000)

// 스케줄 해제
unscheduleHarness(harnessId);

// 현재 활성 스케줄 목록
listScheduledJobs();
```

**주의:** 현재 구현은 `setInterval` 기반의 인-프로세스 스케줄러다. 서버 재시작 시 스케줄이 사라지며, 실제 cron 표현식의 분/시각을 정확히 맞추지 않고 주기(interval)로만 처리한다.

## 실행 상태 관리 (이중 레이어)

| 레이어 | 역할 | 장애 허용 |
|---|---|---|
| 파일 스토어 (`data/runs.json`) | 영구 감사 기록, 모든 상태 기록 | - |
| Redis (`hc:active` Set) | 실시간 "실행 중" 상태, TTL 30분 | 비치명적 (Redis 다운 시에도 실행 계속) |

Redis에서 사라진 run이 파일 스토어에 `running`으로 남아있으면 `failed`로 자동 처리한다 (`recoverOrphanedRuns()`).
