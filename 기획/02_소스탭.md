# 02. 소스 탭 (Sources)

## 개요

소스 탭은 에이전트가 데이터를 읽어올 외부 시스템을 등록·관리하는 화면이다. 현재 PostgreSQL 연결만 활성화되어 있으며, Redis·CSV/Excel·REST API는 UI에 "준비 중" 상태로 표시된다.

---

## 지원 소스 종류

| 소스 | 아이콘 | 상태 |
|---|---|---|
| PostgreSQL | 🐘 | 활성 (내장 pg-sandbox 자동 연결) |
| Redis | 🔴 | 준비 중 |
| CSV / Excel | 📄 | 준비 중 |
| REST API | 🌐 | 준비 중 |

---

## pg-sandbox (내장 소스)

현재 운영 중인 유일한 소스는 `pg-sandbox`로, 코드에 하드코딩된 금융 도메인 PostgreSQL 데이터베이스다. 사용자 설정 없이 환경 변수(`PG_HOST`, `PG_PORT`, `PG_DB`, `PG_USER`, `PG_PASSWORD`)로 연결된다.

**소스 메타데이터 (레지스트리 응답):**

```json
{
  "id": "pg-sandbox",
  "name": "pg-sandbox",
  "type": "postgresql",
  "description": "금융 도메인 PostgreSQL DB (customers, transactions, loans, investments)",
  "status": "connected",
  "tables": ["customers", "transactions", "accounts", "loans", "investments"]
}
```

**실제 연결 정보 (환경 변수 기본값):**

| 환경 변수 | 기본값 |
|---|---|
| `PG_HOST` | `localhost` |
| `PG_PORT` | `5499` |
| `PG_DB` | `sandbox` |
| `PG_USER` | `sandbox` |
| `PG_PASSWORD` | `sandbox` |

## ISource 인터페이스

모든 소스는 `lib/types.ts`의 `ISource`를 구현해야 한다.

```typescript
interface ISource {
  readonly id: string;
  readonly name: string;
  readonly type: string;
  readonly description: string;
  getTools(): ITool[];     // 이 소스에서 제공하는 도구 목록
  ping(): Promise<boolean>; // 연결 상태 확인
}
```

---

## PostgreSQLSource 구현

`lib/sources/postgresql.ts`에 구현되어 있다.

```typescript
class PostgreSQLSource implements ISource {
  private readonly pool: pg.Pool;  // max: 3, idleTimeout: 30s, connTimeout: 5s

  getTools(): ITool[] {
    return [
      GetSchemaTool,       // get_schema
      ExecuteQueryTool,    // execute_query
      ExplainQueryTool,    // explain_query
      GetTableSampleTool,  // get_table_sample
      GetStatsTool,        // get_stats
    ];
  }

  async ping(): Promise<boolean> {
    // SELECT 1 로 연결 테스트
  }

  async close(): Promise<void> {
    // 풀 종료 (executor에서 finally 블록에서 호출)
  }
}
```

**커넥션 풀 설정:**
- 최대 연결 수: 3
- 유휴 타임아웃: 30초
- 연결 타임아웃: 5초

## 소스 연결 확인 흐름 (Orchestrator)

`Orchestrator.run()` 시작 시 등록된 모든 소스에 대해 `ping()`을 병렬로 실행한다. 응답이 성공한 소스만 도구 레지스트리에 등록되어 에이전트가 사용할 수 있다.

```typescript
const available = await Promise.allSettled(
  sources.map(async s => ({ source: s, ok: await s.ping() }))
);
// ok === true인 소스만 사용
```

SSE로 `{ type: "source_check", message: "소스 연결 확인 중..." }` 이벤트가 전송되고, 이후 `{ type: "source_selected", sources: [...] }` 이벤트로 사용 가능한 소스 목록이 전달된다.

## UI — PostgreSQL 연결 폼

소스 탭 하단에 PostgreSQL 연결 폼이 있다. 현재 UI만 존재하며, 실제 저장/연결 테스트 로직은 미구현(준비 중) 상태다.

**폼 필드:** Host, Port, Database, Username, Password

## 스키마 정보 (pg-sandbox 데이터베이스)

에이전트가 `get_schema` 도구를 통해 확인하는 테이블 구조:

| 테이블 | 규모 | 주요 용도 |
|---|---|---|
| `customers` | 10만 건 | 고객 정보, 등급(GOLD/PLATINUM/DIAMOND) |
| `transactions` | 200만 건 | 거래 내역, 금액, 일시 |
| `accounts` | - | 계좌 정보 |
| `loans` | - | 대출 상품, 상태, 연체 |
| `investments` | - | 투자 포트폴리오, 자산 유형별 현황 |
